# 博客系统需求分析与功能模块划分

## 一、项目概述

### 1.1 项目背景

本项目是一个全栈博客系统，采用前后端分离架构，包含三个独立子系统：

| 子系统 | 技术栈 | 说明 |
|--------|--------|------|
| Frontend | Next.js 15 + shadcn/ui | 博客前台，面向访客 |
| Admin-Frontend | Vite + React + shadcn-admin | 管理后台，面向管理员 |
| Backend | FastAPI + MySQL | RESTful API 服务 |

### 1.2 目标用户

- **访客**：浏览文章、搜索内容、评论互动
- **作者**：发布文章、管理内容、查看数据
- **管理员**：用户管理、系统配置、内容审核

## 二、功能需求分析

### 2.1 用户系统模块

#### 2.1.1 用户角色设计

| 角色 | 权限说明 |
|------|----------|
| Guest（访客） | 浏览公开文章、搜索、评论（需登录） |
| Author（作者） | 发布/编辑自己的文章、管理评论 |
| Admin（管理员） | 全部权限，包括用户管理、系统配置 |

#### 2.1.2 认证功能

| 功能 | 前端 | 管理端 | 后端 API |
|------|------|--------|----------|
| 用户注册 | ✅ | - | POST /api/v1/auth/register |
| 用户登录 | ✅ | ✅ | POST /api/v1/auth/login |
| 退出登录 | ✅ | ✅ | POST /api/v1/auth/logout |
| 密码重置 | ✅ | ✅ | POST /api/v1/auth/forgot-password |
| Token 刷新 | ✅ | ✅ | POST /api/v1/auth/refresh |
| OAuth 登录 | 可选 | - | GET /api/v1/auth/oauth/{provider} |

#### 2.1.3 用户信息管理

| 功能 | 前端 | 管理端 | 后端 API |
|------|------|--------|----------|
| 查看个人资料 | ✅ | ✅ | GET /api/v1/users/me |
| 修改个人资料 | ✅ | ✅ | PUT /api/v1/users/me |
| 修改密码 | ✅ | ✅ | PUT /api/v1/users/me/password |
| 上传头像 | ✅ | ✅ | POST /api/v1/users/me/avatar |

### 2.2 文章管理模块

#### 2.2.1 文章核心功能

| 功能 | 前端 | 管理端 | 后端 API |
|------|------|--------|----------|
| 文章列表（分页） | ✅ | ✅ | GET /api/v1/articles |
| 文章详情 | ✅ | ✅ | GET /api/v1/articles/{id} |
| 创建文章 | - | ✅ | POST /api/v1/articles |
| 编辑文章 | - | ✅ | PUT /api/v1/articles/{id} |
| 删除文章 | - | ✅ | DELETE /api/v1/articles/{id} |
| 批量操作 | - | ✅ | POST /api/v1/articles/batch |
| 文章搜索 | ✅ | ✅ | GET /api/v1/articles?keyword=xxx |

#### 2.2.2 文章属性设计

```python
class Article:
    id: int                    # 文章ID
    title: str                 # 标题
    slug: str                  # URL 别名（SEO友好）
    summary: str               # 摘要
    content: str               # 正文内容（Markdown）
    cover_image: str           # 封面图片
    category_id: int           # 分类ID
    author_id: int             # 作者ID
    status: ArticleStatus      # 状态：draft/published/archived
    is_top: bool               # 是否置顶
    is_featured: bool          # 是否推荐
    view_count: int            # 浏览量
    like_count: int            # 点赞数
    comment_count: int         # 评论数
    published_at: datetime     # 发布时间
    created_at: datetime       # 创建时间
    updated_at: datetime       # 更新时间
```

#### 2.2.3 文章状态流转

```
草稿(draft) ──发布──> 已发布(published) ──归档──> 已归档(archived)
     │                    │                      │
     └────────────────────┴──────────────────────┘
                    可恢复为草稿
```

### 2.5 搜索功能模块

#### 2.5.1 搜索功能

| 功能 | 说明 | 实现方式 |
|------|------|----------|
| 全文搜索 | 标题、内容关键词搜索 | 数据库 LIKE 或 Elasticsearch |
| 分类筛选 | 按分类过滤文章 | SQL WHERE |
| 标签筛选 | 按标签过滤文章 | 关联查询 |
| 时间筛选 | 按发布时间范围筛选 | SQL BETWEEN |
| 排序 | 按时间、浏览量、点赞数排序 | SQL ORDER BY |

#### 2.5.2 搜索 API 设计

```
GET /api/v1/search?
    keyword=xxx&
    category=1&
    tags=1,2,3&
    start_date=2024-01-01&
    end_date=2024-12-31&
    sort_by=published_at&
    sort_order=desc&
    page=1&
    page_size=10
```

## 五、API 接口规范

### 5.1 接口命名规范

- 使用 RESTful 风格
- 资源名使用复数形式
- 版本号放在 URL 中：`/api/v1/`

### 5.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

### 5.3 分页响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [ ... ],
    "total": 100,
    "page": 1,
    "page_size": 10,
    "total_pages": 10
  }
}
```

### 5.4 错误响应格式

```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

## 七、技术实现要点

### 7.1 后端技术要点

- 使用 SQLAlchemy 异步 ORM
- JWT Token 认证
- 密码使用 bcrypt 加密
- 文件上传使用本地存储或云存储
- API 文档使用 Swagger/OpenAPI

### 7.2 前端技术要点

- 响应式设计，支持移动端
- SEO 优化（SSG/SSR）
- 图片懒加载
- 代码分割优化加载速度

### 7.3 管理端技术要点

- 富文本编辑器（Markdown）
- 图片上传预览
- 数据表格分页、排序、筛选
